Architecture Plan

The best of
four codebases.

Four open-source projects independently solved different parts of the AI coding agent problem. We studied all of them. Now we're combining the best of each into one system.

AI coding agents can write code.
But can you actually trust and ship what they produce?

Four hard problems.

Trust
Agents claim they did the work.
Did they? Hallucinations are invisible.
Scale
Parallel agents, parallel branches.
Merging is where it all breaks.
DevOps
CI fails. Reviewers comment.
Who tells the agent? You. Every time.
Tenacity
An agent that gives up isn't useful.
Real work means persistent PRs, not one-shots.

We found four projects that each solved one of these problems.

Each made a
different bet.

Correctness

tcagent

Wraps any coding agent in a quality-enforcement layer. 7-step checklist. Two-tier hallucination detection. Tool approval gates. Reviewer agent. Full SQLite audit trail. 726 tests.
github.com/TaisoAI/tcagent
Speed

Longshot

Decomposes a prompt into hundreds of tasks across 50 parallel sandboxes. Self-healing merge pipeline. Scope tracking. Reconciler with adaptive sweeps. Stateless — Git is the only state.
github.com/Blastgits/longshot
Coordination

Agent Orchestrator

Fleet manager with 7-slot plugin architecture. Background lifecycle worker reacts to CI failures, review comments, merge conflicts. Notification routing. Orchestrator is itself an AI agent.
github.com/ComposioHQ/agent-orchestrator
Persistence

pi-autoresearch

Autonomous optimization loop in 1,630 lines. No loop in code — emergent from prompt engineering. Append-only JSONL survives any crash. Auto-resume across context boundaries.
github.com/davebcn87/pi-autoresearch

Unique Contributions

Patterns worth extracting.

From tcagent
Veracity checking
Tier 1: regex catches hallucinated execution (zero cost). Tier 2: LLM grounding check identifies unsubstantiated claims.
Tool policy
4 tiers — auto_approve, evaluate (LLM decides), require_human (blocks until /approve), blocked (terminates). Glob matching.
Reviewer agent
Second Claude instance, read-only tools. Verdicts: ON_TRACK, ISSUES_FOUND, NEEDS_USER. Per-step, not post-hoc.
Audit trail
SQLite with 7 tables. Every conversation turn, tool decision, and veracity check is recorded and queryable.
From Longshot
Task decomposition
Root planner → subplanner (depth 3). Delta-optimized replanning saves ~40K chars per cycle. Persistent LLM conversation.
Merge pipeline
Three layers: merge queue (rebase+retry), reconciler (adaptive tsc+build+test sweeps), finalization (3 corrective cycles).
LLM routing
Multi-endpoint weighted round-robin. EMA latency tracking (α=0.3). Circuit breaker at 3 failures. Recovery probe at 30s.
Dependency injection
Every major class accepts a *Deps interface. Tests inject fakes for spawn, sleep, time, filesystem. Deterministic and fast.
From Agent Orchestrator
Plugin architecture
7 slots: Runtime, Agent, Workspace, Tracker, SCM, Notifier, Terminal. Dynamic import. Graceful degradation.
Reaction engine
CI failed → send to agent (2 retries, escalate). Changes requested → forward (30 min escalation). Configurable per project.
SCM integration
CI checks, review decisions, merge readiness, bot detection, webhook verification. Full DevOps lifecycle awareness.
Orchestrator-as-agent
Supervisor is itself a Claude Code session with CLI access. Recursive agent-manages-agents with read-only constraints.
From pi-autoresearch
Emergent loop
No loop in code. Emergent from: skill rules ("LOOP FOREVER"), system prompt injection, auto-resume on context exhaustion.
Append-only JSONL
State never mutated. Config headers create segments. reconstructState() replays log on startup. Survives any crash.
Auto-resume
Context exhaustion → 5-min cooldown → sendUserMessage(). Turn cap: 20. Fresh agent reads persistent state and continues.
Metric-gated commits
Only commit if metric improved AND checks pass. Extension handles commits; agent handles reverts. Prevents bad code from landing.

Technical Decisions

Five key choices.

1

TypeScript over Python

3 of 4 projects are TypeScript. Better async model, native ESM plugin imports, shared types across CLI/web/mobile, mature monorepo tooling. Port tcagent's patterns.

2

Flat files over databases

All four projects avoid databases. Session metadata as key=value files. Events as append-only JSONL. Config as Zod-validated YAML. No daemon, no migrations.

3

Polling first, webhooks later

30-second polling is simpler and needs no infrastructure. Add webhooks for latency-sensitive CI/review events after the foundation is solid.

4

Quality pipeline is optional

Each stage independently toggleable. Default: Tier 1 veracity (zero-cost regex) + auto-detected quality gates. Reviewer agent is opt-in.

5

Parallelism is opt-in

Sequential single-session is the default. Decomposition requires --decompose flag. No merge queue overhead for simple tasks.

What we're not building.

Deliberate exclusions. Saying no is as important as saying yes.

× Self-hosted LLM infrastructure. Support any OpenAI-compatible endpoint instead.
× Custom TUI framework. Use tmux attach and the web dashboard.
× Agent-internal tool implementation. Manage agents, don't replace them.
× Complex intent classification. CLI commands are explicit. Keep it simple.
Risk Management

What could go wrong.

Complexity explosion
Phase-gated delivery. Each phase ships independently. Phase 1–2 before 3–4.
Quality pipeline too slow
Every stage toggleable. Default: lightweight Tier 1 + auto-detect gates.
Parallel merge conflicts
Scope tracker prevents overlap. Merge queue serializes. Reconciler catches the rest.
Agent CLI updates break plugins
Stable plugin interface. Agent-specific parsing encapsulated in the plugin.
File state corruption
Atomic writes (temp+rename). O_EXCL for creation. JSONL append-only. Single writer.
Auto-resume infinite loops
5-min cooldown. Turn cap (20). Must have done work before resuming.
Orchestrator bad decisions
Read-only constraints. All code changes go through workers with quality pipelines.
Milestones

How we'll know
it's working.

Phase 1

Spawn, message, and kill isolated agent sessions with any CLI. Clean plugin architecture. Tests pass with injected deps.

Phase 2

Hallucinations caught before commit. Dangerous tools gated. Quality gates run automatically.

Phase 3

CI failures auto-retry. Reviews auto-forward. Context auto-resumes. Humans notified only for escalation.

Phase 4

20-file feature → 4-5 parallel sessions. Verified. Merged cleanly. Build passes after integration.

Phase 5

Full cost observability. Experiment mode. Orchestrator agent. Self-healing recovery.

And one more thing.

The orchestrator
is itself an agent.

The supervisory orchestrator is a Claude Code session that uses CLI commands to manage worker agents. It reasons about task assignment, priority, and coordination at a higher level than any individual worker. Agents managing agents.
